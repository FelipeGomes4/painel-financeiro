🔥 Firebase Setup

Configurar
Sincronização

Configure o Firebase para sincronizar os dados do Painel Financeiro em todos os seus dispositivos.

⏱️ Tempo estimado: 15 minutos
🔥

Parte 1 — Criar projeto no Firebase

Gratuito — plano Spark cobre tranquilamente uso pessoal

1

Acesse console.firebase.google.com

Entre com sua conta Google. Clique em "Criar um projeto".

2

Nomeie o projeto

Use um nome como painel-financeiro. Desative o Google Analytics (não é necessário). Clique em Criar projeto.

3

Adicione um app Web

Na página do projeto, clique no ícone </> (Web). Dê o nome painel-financeiro-web. Não ative o Firebase Hosting. Clique em Registrar app.

Uma tela com as configurações aparecerá — copie o objeto firebaseConfig. Ficará assim:

const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "meu-projeto.firebaseapp.com", projectId: "meu-projeto", storageBucket: "meu-projeto.appspot.com", messagingSenderId: "123456789", appId: "1:123:web:abc123" };
⚠️
Guarde essas informações — você vai precisar colar no arquivo index.html no próximo passo.
🔐

Parte 2 — Ativar autenticação

Para login com e-mail e Google

1

Vá em Authentication

No menu lateral do Firebase Console, clique em Build → Authentication. Clique em "Começar".

2

Ative E-mail/Senha

Na aba Sign-in method, clique em E-mail/Senha, ative a primeira opção e salve.

3

Ative Google (opcional)

Clique em Google, ative, informe seu e-mail de suporte e salve. Isso permite login com 1 clique.

💡
Se hospedar no GitHub Pages, adicione o domínio em Authentication → Settings → Authorized domains: SEU-USUARIO.github.io
🗄️

Parte 3 — Criar banco de dados Firestore

Onde os dados financeiros serão armazenados

1

Vá em Firestore Database

No menu lateral, clique em Build → Firestore Database. Clique em "Criar banco de dados".

2

Escolha as configurações

Localização: southamerica-east1 (São Paulo — mais rápido no Brasil). Modo: Produção. Clique em Criar.

3

Configure as regras de segurança

Clique na aba Regras e substitua o conteúdo por:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Cada usuário só acessa seus próprios dados match /users/{userId}/{document=**} { allow read, write: if request.auth != null && request.auth.uid == userId; } } }

Clique em Publicar.

✅
Seguro! Esta regra garante que cada usuário só acessa os próprios dados — ninguém vê as finanças de outra pessoa.
⚙️

Parte 4 — Colar as configurações no app

Última etapa — conectar o app ao Firebase

1

Abra o arquivo index.html

Abra o index.html em qualquer editor de texto (Notepad, VS Code, etc.).

2

Encontre o bloco FIREBASE_CONFIG

Use Ctrl+F (ou Cmd+F) e busque por:

COLE_AQUI_apiKey

Você verá este bloco:

const FIREBASE_CONFIG = { apiKey: "COLE_AQUI_apiKey", authDomain: "COLE_AQUI_authDomain", projectId: "COLE_AQUI_projectId", storageBucket: "COLE_AQUI_storageBucket", messagingSenderId: "COLE_AQUI_messagingSenderId", appId: "COLE_AQUI_appId" };
3

Substitua pelos seus valores

Cole os valores do Firebase Console que você copiou na Parte 1:

const FIREBASE_CONFIG = { apiKey: "AIzaSyXXXXXXXXXXXXXXX", ← seu valor authDomain: "meu-projeto.firebaseapp.com", projectId: "meu-projeto", storageBucket: "meu-projeto.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef" };

Salve o arquivo.

4

Faça upload do index.html atualizado no GitHub

No repositório GitHub, clique no arquivo index.html → ícone de lápis ✏️ → Selecione tudo (Ctrl+A) → Cole o novo conteúdo → Commit changes.

O deploy roda automaticamente em ~1 minuto.

✅
Pronto! Ao abrir o app, uma tela de login aparecerá. Crie uma conta, entre e os dados sincronizam automaticamente em todos os dispositivos.
